# chkLink 使用手冊

## 這是什麼

`chkLink` 是一套 Windows 網站失效連結掃描工具，提供圖形介面（GUI）與命令列（CLI）兩種使用方式。

一般使用者通常只需要使用 GUI。它可以幫你：

- 掃描網站中的站內連結是否可正常開啟
- 列出頁面中的外部連結
- 檢查圖片是否缺少 `alt` 屬性
- 提醒頁面內是否仍有 `http://` 連結
- 輸出 Excel 報告與 log 紀錄

## 安裝方式

一般情況下，請使用安裝程式版本：

- `chklink-<version>-win-x64-setup.exe`

安裝完成後：

- GUI 會是主要入口
- CLI 會一併安裝，但通常不需要手動使用
- 使用者自己的設定與快取會放在你的 Windows 使用者資料夾裡

## 第一次使用

第一次啟動後，程式會在需要時建立這些檔案：

- `config.yaml`
- `visited_link.yaml`

其中：

- `config.yaml` 是你的設定檔
- `visited_link.yaml` 是已檢查過且回應正常的連結快取

這兩個檔案通常會放在：

- `C:\Users\你的帳號\AppData\Local\chkLink\data\`

如果你看到文件寫 `%LOCALAPPDATA%\chkLink\data\`，它指的就是這個位置。  
例如你的 Windows 帳號如果是 `peter`，實際路徑通常會像這樣：

- `C:\Users\peter\AppData\Local\chkLink\data\`

## 基本操作流程

1. 開啟 GUI。
2. 在 `網址掃描` 分頁的 `URLs` 區塊輸入要掃描的網址。
3. 如果需要，切到 `系統設定` 分頁調整掃描設定。
4. 回到 `網址掃描` 分頁，按 `掃描`。
5. 等待掃描完成。
6. 若畫面詢問是否開啟檔案所在目錄，可直接選擇開啟。
7. 查看 Excel 報告與 log 紀錄。

## 依畫面操作的方式

### 1. 在哪裡輸入網址

主畫面第一個分頁是：

- `網址掃描`

在這個分頁中：

- `URLs` 區塊：輸入想要掃描的網址
- 若有多個網址，可用逗號分隔
- `掃描` 按鈕：開始掃描
- `離開` 按鈕：未掃描時離開程式
- 掃描進行中，`離開` 會切換成 `中斷`
- `跳過已檢查過的網址`：勾選時，可利用已檢查過的正常連結快取，減少重複檢查的時間
- `掃描歷史`：顯示目前掃描過程與結果訊息

### 2. 在哪裡更改設定

第二個分頁是：

- `系統設定`

這個分頁裡常見欄位包括：

- `檢查連結的層數`
- `圖片是否必須有 alt 屬性`
- `檢查不安全的 http 協定`
- `連線逾時秒數`
- `報告路徑`
- `請求的標頭`
- `想要避免檢查的網址清單`
- `想要檢查的網址清單`

如果你改了設定，要按：

- `儲存設定`

按下後，設定才會寫入 `config.yaml`。

### 3. 怎麼改報告儲存位置

在 `系統設定` 分頁中：

1. 找到 `報告路徑`
2. 點旁邊的資料夾按鈕
3. 選擇你要存放報告的資料夾
4. 按 `儲存設定`

之後新的掃描報告就會存到這個位置。

### 4. 怎麼檢查有沒有新版本

在 `系統設定` 分頁中有：

- `檢查更新`

按下後，程式會：

1. 檢查伺服器上的版本
2. 比對你目前安裝的版本
3. 如果有新版本，詢問你是否下載並啟動新版安裝程式

如果你同意更新，程式會下載新版安裝程式，接著交給 installer 覆蓋安裝。

### 5. 掃描完成後會發生什麼事

掃描完成後：

- `掃描歷史` 會顯示完成訊息
- 程式會詢問你是否要開啟檔案所在目錄
- 成功連結快取會嘗試寫回 `visited_link.yaml`

如果快取儲存失敗，畫面仍會先恢復正常狀態，並顯示提示訊息。

## 掃描結果會看到什麼

掃描完成後，通常會得到：

- `.xlsx` 報告
- `.log` 紀錄

報告內容會依掃描結果列出：

- 失效的站內連結
- 重新導向狀態
- 外部連結列表
- 缺少 `alt` 的圖片
- 其他檢查結果

## 掃描範圍重點

`chkLink` 目前的設計重點是「本站內容維護」：

- 站內連結會驗證，也會依設定繼續遞迴掃描
- 外部連結只列出，不會對外站做遞迴驗證
- 外部連結若是 `http://`，可列為內容維護提醒

這樣可以避免把外站健康檢查混進本站維護工作中。

## 設定檔在哪裡

實際使用的設定檔通常位於：

- `C:\Users\你的帳號\AppData\Local\chkLink\data\config.yaml`

常見可調整內容包括：

- 掃描深度
- 報告輸出位置
- 是否跳過已檢查過的網址
- `headers`
- `avoid_urls`

如果某些網站需要登入態、SSO 或特殊 header，通常也是在這裡調整。

## 已檢查過的網址快取

成功連結快取通常位於：

- `C:\Users\你的帳號\AppData\Local\chkLink\data\visited_link.yaml`

它的用途是把先前檢查過、而且回應正常的連結記錄下來，避免之後又對同一批正常網址重複偵測，進而增加掃描時間。  
但這個效果有一個前提：

- `網址掃描` 分頁中的 `跳過已檢查過的網址` 必須保持勾選

如果你取消勾選：

- 程式仍可能保留 `visited_link.yaml`
- 但掃描時就不會用它來跳過先前已確認正常的網址
- 掃描時間也可能因此變長

## 更新方式

目前 `chkLink` 的 GUI 更新流程是走 installer 覆蓋安裝，不是直接替換單一 `exe`。

這代表：

- 更新時會下載新版安裝程式
- 再由安裝程式覆蓋安裝
- `config.yaml` 與 `visited_link.yaml` 會保留

## 常見提示

### SmartScreen 或 Smart App Control 提示

首次下載或執行安裝程式時，Windows SmartScreen 或 Smart App Control 仍可能出現提示。這不一定代表檔案有問題，而是 Windows 防護機制對新下載程式的常見提醒。

### 安裝在 `Program Files` 會不會影響使用

目前 `1.4.1` 起，使用者資料已改放到你的 Windows 使用者資料夾中，因此就算程式安裝在 `Program Files`，一般使用者也不需要把設定與快取寫回安裝目錄。

### 我怎麼找到設定檔資料夾

如果你要找設定檔或快取資料夾，可用下列方式：

1. 在 Windows 檔案總管的路徑列輸入 `%LOCALAPPDATA%\chkLink\data`
2. 或直接前往：
   `C:\Users\你的帳號\AppData\Local\chkLink\data\`

如果看不到 `AppData`，通常是因為它是隱藏資料夾，這是 Windows 的正常行為。

## 什麼情況需要找維護者

遇到下列情況時，建議找維護者協助：

- 更新按鈕無法下載新版
- 特定站台需要特殊登入態或自訂 header
- 掃描結果與預期差異很大
- 需要使用 CLI 做排程或自動化
- 需要建置、簽章、GitHub Release 或 winget 發佈
